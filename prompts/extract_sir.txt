Sei un analista OSINT/data-journalism.
Analizza questo documento PDF contenente uno o più Serious Incident Reports Frontex.

Obiettivo:
1) quantificare vittime per ciascun SIR;
2) definire in modo strutturato il luogo in cui avviene l'incidente.
3) classificare SOLO record che sono realmente SIR (con o senza ID numerico).

Restituisci SOLO JSON valido, senza markdown, con questa struttura:
{
  "records": [
    {
      "sir_id": "12345/2021",
      "report_date": "YYYY-MM-DD or null",
      "incident_date": "YYYY-MM-DD or null",
      "location_details": "string or null",
      "where_clear": "short clear location statement in English or null",
      "location_text_raw": "short raw location wording from the document or null",
      "country_or_area": "country, SAR/SRR area, island, hotspot, camp, sea area, or null",
      "location_type": "sea|land|facility|mixed|unknown or null",
      "precision_level": "exact|approximate|broad|unknown or null",
      "geocodable": "yes|no or null",
      "geocodable_query": "best query string to geocode this location, or null",
      "lat": "number or null",
      "lon": "number or null",
      "uncertainty_note": "why location is uncertain, or null",
      "dead_confirmed": 0 or null,
      "injured_confirmed": 0 or null,
      "missing_confirmed": 0 or null,
      "dead_possible_min": 0 or null,
      "dead_possible_max": 0 or null,
      "possible_violations": [
        {
          "violation_name": "string",
          "legal_basis": "string or null",
          "assessment": "likely|possible|unclear|not_stated"
        }
      ],
      "context_note": "string or null",
      "libyan_coast_guard_involved": true or false or null,
      "evidence_quote": "short quote from the document",
      "confidence": "high|medium|low",
      "evidence_pages": [1, 2]
    }
  ]
}

Regole:
- Identifica ogni blocco "Serious Incident Report" (con o senza numero).
- Prima fase di detection: cerca sezioni che iniziano con "Serious Incident Report" o equivalente.
- Se nel documento non ci sono SIR, restituisci `{"records":[]}`.
- Dai priorità ai campi Dead/Injured/Missing persons nel form.
- Se i campi non hanno numeri, usa Details / Information/Allegations / Assessment.
- Non inventare numeri.
- Distingui confermato vs possibile/non confermato.
- Se impossibile estrarre un valore, usa null.
- sir_id: usa il numero del SIR nel formato in cui appare nel documento.
  - Se il formato è `XXXXX/YYYY`, usalo così (es. `"11013/2019"`).
  - Se è solo numerico senza anno (es. `no. 911`), costruisci `"911/<anno_da_report_date>"` (es. `"911/2019"`).
  - Se il numero è assente o illeggibile, usa null.
- Non usare ID operativi o testuali (es. `POSEIDON_001/2021`, `THEMIS_001/2021`): questi NON sono SIR validi per questo output.
- evidence_pages: numeri di pagina del PDF dove hai trovato le informazioni.
- libyan_coast_guard_involved: true se il documento menziona esplicitamente la guardia costiera libica (Libyan Coast Guard, LCG, guardia costiera della Libia) come attore presente o coinvolto nell'incidente; false se l'incidente è descritto senza alcun coinvolgimento libico; null se non è possibile determinarlo.
- Non inventare coordinate: `lat` e `lon` devono essere null a meno che siano esplicitamente presenti nel documento.
- Se coordinate esplicite sono presenti in gradi decimali, riporta solo numeri decimali (es. `35.12345`, `24.98765`).
- Non usare formati con `N/E/S/W`, simboli di grado o stringhe testuali nelle coordinate.
- Se le coordinate non sono in decimali o non sono chiaramente convertibili, usa `lat=null` e `lon=null`.
- Se il report contiene la sezione "Possible violation(s) of fundamental rights enquired", estrai OGNI voce in `possible_violations` come elemento separato.
- `possible_violations` deve essere `[]` se la sezione non e presente.
- `assessment` in `possible_violations` deve essere normalizzato a: `likely|possible|unclear|not_stated`.
- Se non c'e valutazione esplicita per una voce, usa `not_stated`.

Rubrica confidence (stretta):
- high: luogo esplicito e chiaramente identificabile (o area marittima ben delimitata) con evidenza testuale diretta.
- medium: luogo parzialmente inferito dal contesto (operazione, città vicina, soggetti coinvolti), con ambiguità moderata.
- low: luogo generico/redatto/ambiguo o con indizi confliggenti.

Regola geocodable:
- `yes` solo se è possibile costruire una query cartografica praticabile e relativamente delimitata.
- `no` se il luogo è troppo generico/redatto/ambiguo per geocodifica affidabile.
- Se `geocodable` è `no`, `geocodable_query` deve essere null.
